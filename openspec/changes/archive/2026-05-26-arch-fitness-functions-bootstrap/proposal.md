# arch-fitness-functions-bootstrap

## Why

Per ADR-0001 (Pattern 3), architectural rules expressed as CLAUDE.md prose rot the moment the next contributor diverges. The audit found at least one such drift today (`CLAUDE.md` claims "four canonical sites use wobble"; reality is two). The pattern that fixes the class is **every "Never" rule has a CI test**.

This change installs the two tools that ADR-0001 chose (Tach for package boundaries; pytest-archon for AST/call-site rules), grandfathers existing violations via `tach sync`, and lands the initial archon rule set — including the `json.loads`-ban that backstops `wobble-typed-funnel`.

The Tach spike (2026-05-26) caught 38 violations in ~30 seconds, including:

- 23 instances of domain reaching into package submodules (e.g. `llm_resource.py:78` → `providers.anthropic`).
- 1 cross-package coupling missed by the original audit: `block_detector → packages.escalation`.
- A handful of subtle "submodule reaches back to parent's `errors` / `prompts`" smells.

Grandfathering these into `tach.toml`'s violations snapshot (Packwerk-style ratchet) lets the project ship immediately while preventing *new* violations. Real cleanup can be sequenced afterwards.

## What changes

### New dev tooling

```toml
# pyproject.toml [tool.uv.dev-dependencies] (or equivalent)
"tach>=0.20",            # Rust binary; uvx-compatible; brings the [[interfaces]] model
"pytest-archon>=0.0.6",  # archrule fluent API on top of pytest
```

### New top-level `tach.toml`

Generated by `tach sync` after running `tach mod` interactively (or hand-curated from the explore spike). Declares each `packages/*` subpackage as a module with `[[interfaces]]` listing its public exports.

```toml
source_roots = ["src"]
exclude = ["benchmarks", "docs", "eval", "scripts", "tests"]

[[modules]]
path = "a2web"
depends_on = [
    "a2web.packages.llm_extract",
    "a2web.packages.browser_pool",
    # ... full list
]

[[modules]]
path = "a2web.packages.llm_extract"
depends_on = []

# ... one [[modules]] per packages/X subpackage

[[interfaces]]
from = ["a2web.packages.llm_extract"]
expose = [
    # The current __all__ of packages/llm_extract/__init__.py
    "Wobbled", "parse_with_policy", "unwrap", "WobblePolicy", # (post wobble-typed-funnel)
    # ...
]

# ... one [[interfaces]] per package
```

### New `Makefile` target

```makefile
arch: ## Run architectural fitness functions
	uv run tach check
	uv run pytest tests/architecture/ -v

check: lint ty test arch ## Full quality gate
```

`make check` gains a fourth gate. Tach runs in ~10ms; archon tests run as a normal pytest module (~100ms).

### New `tests/architecture/` directory

Replaces the hand-rolled `tests/test_packages_independence.py` (deleted, its rule absorbed into Tach's `depends_on` lists).

Initial archon rules (each in its own test file):

- **`test_json_loads_funnel.py`** — Backstop for `wobble-typed-funnel`. Walks every `.py` under `src/a2web/packages/llm_extract/` and asserts no `json.loads` call outside `wobble/`. AST-based, not text-based.
- **`test_no_dict_str_any_on_dataclasses.py`** — Bans `dict[str, Any]` fields on internal `dataclass(slots=True)` (CLAUDE.md: "Never reintroduce `tier_extras: dict[str, Any]`"). Walks dataclass definitions, inspects field annotations.
- **`test_tools_return_pydantic_not_str.py`** — Walks `@a2kit.read` / `@a2kit.write` decorated functions, asserts return type is a `BaseModel` subclass or `dict`, never `str` (CLAUDE.md: "Don't return `-> str` from a tool").
- **`test_no_lambdas_in_app_provide.py`** — Walks `app.provide(...)` call sites, asserts the argument is a named function, not a lambda (CLAUDE.md: "Never use a lambda in `app.provide`").
- **`test_response_models_at_module_scope.py`** — Walks pydantic `BaseModel` subclasses, asserts they're defined at module top level when used as tool return types (CLAUDE.md: "All return-type pydantic models at module scope").

Existing test_packages_independence.py rule is now redundant with Tach's `packages/X depends_on = []` — delete after Tach lands.

### Grandfathered violations

`tach sync` generates `tach.toml`'s initial state with current violations marked. Each entry in the ignore-list gets a one-line comment pointing at the openspec change that will retire it (most go to `unify-plugin-manifests` for the `providers/` smells).

## Impact

**Code changes**
- Two new dev dependencies. No production runtime impact.
- One new top-level config (`tach.toml`).
- One new test directory (`tests/architecture/`).
- One deletion (`tests/test_packages_independence.py`, after Tach lands).
- `Makefile` updates.

**CI impact**
- `make check` gains ~150ms of wall time (Tach + archon together). Negligible.
- New failure mode: a contributor introducing a layering violation gets a clear `tach check` error pointing at the offending file:line plus the violated interface. The ratchet refuses any new violation; existing ones are grandfathered until retired.

**Documentation**
- CLAUDE.md "Never" list trimmed: each rule now points at the test that enforces it. Format: `Never X — enforced by tests/architecture/test_X.py.`
- New `docs/architecture/README.md` summarising the two tools, the ratchet workflow, and how to add a new rule.

**Risk**
- LOW. Both tools are mature (Tach 2024+, archon 2023+) and well-supported in CI. Tach's `[[interfaces]]` migration from the deprecated `strict_mode` was handled automatically during the spike; no further migration churn expected on our usage.
- ONE risk: false positives from over-eager rules. Mitigation: archon rules ship with explicit allowlists; new rules added cautiously with at least one capability-test pair (rule + counter-example).

**Out of scope (deferred)**
- Retiring grandfathered violations from `tach.toml` — happens organically via `unify-plugin-manifests` (providers cleanup retires ~25 of the 38) and a follow-up `cross-package-coupling-cleanup` change (the `block_detector → escalation` smell).
- Migrating the wobble funnel itself — that's `wobble-typed-funnel`, sequenced ahead of this change. The `json.loads` ban rule depends on the new `wobble/` folder existing.
- Adding `import-linter` as a third tool. Tach + archon cover the rule shapes we have.

# Output benchmark ‚Äî findings, 2026-05-22

First run of the re-runnable output benchmark (`make bench`). Corpus:
`eval/corpus.yaml` ‚Äî 10 URLs across listing / comments / clean / gated / spa.
Provider: `claude-code` (OAuth subscription). Judge: `claude-sonnet-4-6`.
Run: 30 cells, $2.06, 5m20s wall. Run dir (regenerable, gitignored):
`eval/runs/2026-05-22_124002/`.

## Headline ‚Äî four axes, per system

| System | quality | env tokens | clarity | contract | next_links |
|---|---|---|---|---|---|
| webfetch_baseline | 2.90 | 167 | 3.00 | n/a | n/a |
| a2web_detail | 4.10 | 2554 | 1.67 | 10/10 | 4.00 |
| a2web_extract | **4.40** | 430 | **3.67** | 10/10 | 3.67 |

`a2web_extract` (the `ask` tool) is the strongest system: highest answer
quality, highest clarity, full data-contract conformance, and a lean
envelope ‚Äî all four axes at once.

## Where a2web wins

- **Answer quality:** `a2web_extract` +1.5 over WebFetch (4.40 vs 2.90),
  `a2web_detail` +1.2. a2web won 8/10 URLs (extract) and 7/10 (detail);
  WebFetch 5/10.
- **The gated + SPA edge is real.** WebFetch scores **0.0** on both the
  gated class (medium-tag ‚Äî HTTP 403) and the SPA class (react.dev ‚Äî
  HTTP 404 to plain httpx); a2web scores **5.0** on both. WebFetch reach
  6/10 vs a2web 9/10. This is exactly the failure mode a2web exists for.
- **Clarity:** `a2web_extract` 3.67 vs WebFetch 3.00. The server-side
  extracted answer is more directly actionable than WebFetch's raw text.
- **Token cost of `ask` is near-WebFetch.** `a2web_extract`'s lean
  `AskResponse` envelope averages 430 tokens ‚Äî only +263 over WebFetch's
  167 ‚Äî while delivering +1.5 quality. The envelope diet works.
- **Data contract:** 10/10 conformant for both a2web systems, under
  `debug=False` and `debug=True`. The v0.14 deviation/debug rules hold on
  every URL.
- **next_links:** scored 4.00 / 3.67 on listing URLs that produced
  candidates. WebFetch has no equivalent.

## Where a2web loses / needs work

- **`a2web_detail` clarity collapses to 1.67** (vs WebFetch 3.00) and its
  envelope balloons to 2554 tokens. Handing the agent the full page
  markdown is expensive and hard to act on ‚Äî this is the counterfactual
  that justifies `ask` over `fetch_raw`. Detail mode should stay a
  fallback, not a default.
- **Comment threads are weak for everyone.** The `comments` class scores
  ‚â§2.5 across all systems. `reddit-comments` **fetch-errored on both
  a2web systems**: `tier=raw verdict=length_floor`.
  - *Diagnosis (corrected after probe):* the corpus URL
    (`/r/programming/comments/1ka1bxv/‚Ä¶`) is a **deleted Reddit post**, not
    a live thread. The Reddit handler probed `.json` (404) + `old.reddit`
    (404) and correctly returned `not_found`; the raw tier then got a
    200 + ~8 KB SPA shell ‚Üí gate `length_floor`. A live Reddit thread
    fetches cleanly via the same handler (12 KB content, ~700 ms). So this
    is **not** a browser-escalation gap ‚Äî it is a stale corpus fixture.
  - The one real (minor) bug it exposed: the handler's authoritative
    `not_found` was overwritten by the downstream `length_floor`, so the
    agent saw a retry-looking failure instead of "definitively gone." Fixed
    by the `handler-verdict-precedence` change.
- **Corpus fixture weakness.** Two stale corpus URLs inflate the low
  comments-class scores: `reddit-comments` points at a deleted post, and
  `hn-item` points at `id=40000000`, a placeholder id. Swap both for
  known-live discussions before the next run.

## Contract failures

None. Both a2web systems: 10/10 conformant on the debug=False envelope and
10/10 on the debug=True envelope.

## Fetch errors (5 of 30 cells)

- `reddit-listing` √ó webfetch ‚Äî empty model response (Reddit served
  WebFetch a page Haiku could not answer from).
- `reddit-comments` √ó a2web_detail / a2web_extract ‚Äî `length_floor`, raw
  tier. Root cause: the corpus URL is a deleted post (see above) ‚Äî a stale
  fixture, not a product gap.
- `medium-tag` √ó webfetch ‚Äî HTTP 403 (gated).
- `spa-react-dev` √ó webfetch ‚Äî HTTP 404 (SPA, no JS render).

## Takeaways

1. `ask` (server-side extraction) is the right default ‚Äî it wins every
   axis at once and costs ~430 envelope tokens.
2. The benchmark confirms a2web's core thesis: WebFetch fails on gated and
   JS-rendered pages; a2web does not.
3. Follow-up for the backlog: refresh the two stale corpus fixtures ‚Äî
   `reddit-comments` (deleted post) and `hn-item` (placeholder id) ‚Äî with
   known-live discussions. The verdict-precedence bug the deleted post
   surfaced is already fixed (`handler-verdict-precedence`).

## Did the cascade refactor regress anything?

**No.** The change explicitly promised no wire/envelope change, and the
benchmark confirms it: **contract 10/10 for both a2web systems on the
debug=False AND debug=True envelopes** ‚Äî identical to Run 1. The verdict
now derives from `resolve_verdict(observation log)` instead of a mutable
slot; the response shape is unchanged.

- Answer quality is stable within live-network noise: `a2web_detail`
  4.10 ‚Üí 4.50, `a2web_extract` 4.40 ‚Üí 4.30. Reach 10/10 (detail) and
  9/10 (extract).
- **Comments class recovered.** Was ‚â§2.5 in Run 1 (the deleted-post
  fixture). With the live `ddlc78` thread + `id=9224` item, the comments
  class scores **5.00** for `a2web_extract` and **3.50** for detail. The
  Run-1 `reddit-comments` fetch-error is gone ‚Äî it fetches cleanly through
  the Reddit handler. This validates both the corpus refresh and the
  `no_match` split (matched-but-failed handlers now surface real verdicts).
- Gated + SPA edge holds: WebFetch still **0.0** on `medium-tag` (403) and
  `react.dev` (404); a2web **5.0** on both.

## One new soft spot

- **`gh-trending` √ó a2web_extract scored 0** (judge_reached=False), while
  `a2web_detail` handled the same URL at 3.0. The fetch itself was
  contract-conformant (468 envelope tokens, 10/10 contract still holds) ‚Äî
  this is an extraction-quality miss on the GitHub trending listing, not a
  cascade or envelope fault. Single-URL judge miss on a 10-URL live run;
  watch it next run before treating it as a real regression.

## Contract failures

None. Both a2web systems: 10/10 on debug=False and 10/10 on debug=True.

## Takeaway

The `cascade-decision-log` change is output-neutral: same envelope, same
contract conformance, quality stable within noise. The structural win
(verdict-clobber bug class eliminated, escalation policy unified in one
pure planner) carries no output-quality cost. Safe to archive.

Based on the fetched MDN documentation, here's what the Fetch API is used for and its primary entry point:

## Purpose
The Fetch API provides an interface for **fetching resources across the network**. It's a more powerful and flexible replacement for `XMLHttpRequest`.

## Primary Entry Point
The **`fetch()` method** is the primary entry point. It's a global method available in both:
- `Window` context (browser)
- `Worker` context (web workers)

## How It Works
- **Takes one mandatory argument**: the path to the resource you want to fetch
- **Returns a Promise** that resolves to a `Response` object as soon as the server responds with headers (even if it's an HTTP error status)
- **Optionally accepts a second argument**: an `init` options object to configure the request

## Key Concepts
The Fetch API uses core objects like `Request` and `Response`, along with related concepts such as CORS and HTTP Origin header semantics. Once you have a `Response`, there are multiple methods available to handle the response body content.

There's also a newer **`fetchLater()` method** for deferred fetches that can be sent after a delay or when the page closes/navigates away, but `fetch()` remains the standard entry point for immediate resource fetching.

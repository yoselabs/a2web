Based on the fetched MDN documentation, here's what the Fetch API is used for and its primary entry point:

## Purpose
The Fetch API provides an interface for **fetching resources across the network**. It's a more powerful and flexible replacement for the older `XMLHttpRequest` API.

## Primary Entry Point
The **`fetch()` method** is the primary entry point. It's a global method available in both:
- `Window` context (browsers)
- `Worker` context (web workers)

## How It Works
- Takes one mandatory argument: the path to the resource you want to fetch
- Returns a **`Promise`** that resolves to a `Response` object as soon as the server responds with headers (even for HTTP error statuses)
- Optionally accepts a second `init` argument for configuration options

Once you have a `Response`, you can use various methods to define what the body content is and how it should be handled.

**Note:** While you can technically create `Request` and `Response` objects directly using their constructors, this is uncommon. They're typically created as results of other API actions (like `fetch()` calls or service worker `FetchEvent.respondWith()`).

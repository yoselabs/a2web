Based on the Stripe documentation fetched, here's what a PaymentIntent is and its purpose:

## What is a Stripe PaymentIntent?

A **PaymentIntent** is a core API object that represents a payment transaction in Stripe. It encapsulates all the details about a transaction, including:
- The amount to collect
- The desired currency
- Supported payment methods

Each PaymentIntent typically correlates with a single shopping cart or customer session in your application.

## Purpose in the Payment Lifecycle

The PaymentIntent tracks a payment **from creation through checkout** and serves several key functions:

1. **Lifecycle Management** — It maintains a status that changes over the payment's lifecycle, allowing you to monitor the payment from initial creation through completion.

2. **Automatic Authentication Handling** — It automatically triggers additional authentication steps when required (such as 3D Secure or Strong Customer Authentication).

3. **Prevents Common Issues** — It eliminates:
   - Double charges
   - Idempotency key issues

4. **Regulatory Compliance** — It supports Strong Customer Authentication (SCA) and adapts to regulatory changes across different regions.

5. **Payment Attempt Tracking** — A single PaymentIntent may have multiple Charge objects associated with it (for retries or multiple payment attempts), allowing you to inspect the outcome and details of each attempt.

## Key Workflow

Building a PaymentIntent integration involves two main actions:
1. **Creating** the PaymentIntent on your server
2. **Confirming** it (typically on the client side using a client secret)

The PaymentIntent also supports saving payment methods for future use via the `setup_future_usage` parameter, enabling both on-session and off-session payment scenarios.

Based on the Stripe documentation fetched, here's what a PaymentIntent is and its purpose:

## What is a PaymentIntent?

A **PaymentIntent** is a Stripe API object that represents a payment transaction and tracks it through its entire lifecycle—from creation through checkout to completion, including any additional authentication steps required.

## Purpose in the Payment Lifecycle

The PaymentIntent serves several key purposes:

1. **Transaction Tracking**: Each PaymentIntent typically correlates with a single shopping cart or customer session, encapsulating transaction details like:
   - Supported payment methods
   - Amount to collect
   - Desired currency

2. **Status Management**: It maintains a status that changes over the PaymentIntent's lifecycle, allowing you to monitor the payment from start to finish.

3. **Authentication Handling**: It automatically handles complex payment flows, including triggering additional authentication steps when required (like 3D Secure).

## Key Advantages

- **Automatic authentication handling** for regulatory compliance
- **No double charges** through idempotency protection
- **No idempotency key issues**
- **Support for Strong Customer Authentication (SCA)** and regulatory changes

## Workflow

Building a PaymentIntent integration involves two main actions:
1. **Creating** the PaymentIntent on the server
2. **Confirming** it (typically on the client side using the client secret)

The best practice is to create a PaymentIntent as soon as you know the amount (when checkout begins), and reuse the same PaymentIntent if the checkout process is interrupted and resumes later, rather than creating a new one.

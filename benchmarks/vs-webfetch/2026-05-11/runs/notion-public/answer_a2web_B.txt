Based on the Notion Help Center article, here's what a Notion database is and how it differs from a regular page:

## What is a Notion Database?

A Notion database is a **collection of pages** designed to help you manage and organize multiple items in one place. Each item in a database is its own full Notion page that you can build out with text, images, and other content.

## Key Differences from a Regular Page

**1. Multiple items with individual pages**
- A database contains many items, and each item is a separate Notion page
- You can open any database item to add detailed information to it
- Regular pages are typically standalone documents

**2. Customizable properties**
- Databases let you add properties (like dates, status, links) to contextualize and label each item
- Regular pages don't have this structured property system

**3. Multiple visualization options**
- Databases can be viewed as tables, lists, calendars, charts, galleries, and more
- You can switch between different views of the same data
- Regular pages have a single linear format

**4. Linked data across databases**
- You can create relationships between items across multiple databases
- For example, a single database can combine contacts, companies, and deals all in one place while keeping them as separate data sources
- Regular pages don't have built-in relational capabilities

**5. Organizational flexibility**
- Databases support filtering, sorting, and grouping of items
- You can display different properties in different views
- Regular pages are static in their organization

In essence, a database is a **structured, relational way to organize multiple pages**, while a regular page is a **standalone document** without properties or built-in organization features.

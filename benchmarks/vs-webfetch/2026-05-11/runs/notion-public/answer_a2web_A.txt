## What is a Notion Database?

A Notion database is **a collection of pages** organized in a structured way. According to the documentation, databases are one of Notion's fundamental features designed to help you manage and organize multiple pages in one place.

## Key Differences from a Regular Page

**1. Every item is its own page**
- Each entry in a database is a full Notion page that you can open, edit, and add rich content to (text, images, etc.)
- Regular pages are individual documents without this nested structure

**2. Customizable properties**
- Databases allow you to add properties (like dates, status, links, tags) to contextualize and label each item
- Regular pages don't have this structured property system

**3. Multiple visualization options**
- Databases can be displayed in different views: table, list, calendar, board, chart, gallery, and more
- A regular page has only one static format

**4. Ability to link multiple data sources**
- You can view multiple sets of data in a single database simultaneously
- For example, you can create an all-in-one CRM where contacts, companies, and deals coexist but remain separate data sources
- Regular pages are standalone

**5. Structured organization**
- Databases excel at managing many similar items with consistent structure
- Regular pages are better for freeform content

In essence, a database transforms Notion from a document tool into a structured data management system, while still maintaining the flexibility of pages underneath.

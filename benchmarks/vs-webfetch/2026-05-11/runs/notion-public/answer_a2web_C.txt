**What a Notion database is:**
A Notion database is a collection of pages that helps you manage and organize multiple related items in one place.

**How it differs from a regular Notion page:**

1. **Multiple pages vs. single page** — While a regular Notion page is a single document, a database contains multiple pages. Every item you add to a database is its own Notion page that you can open and edit independently.

2. **Customizable properties** — Database items can have customizable properties (dates, status, links, etc.) that let you label and organize each item with metadata. Regular pages don't have this structured property system.

3. **Multiple visualization options** — Database data can be organized and viewed in different formats: table view, list, calendar, chart, gallery, or board. A regular page has a single linear layout.

4. **Linking related data** — Databases let you view multiple sets of related data in one place. For example, you can build an all-in-one CRM database where contacts, companies, and deals are managed together while being treated as separate data sources.

In essence: a regular Notion page is a standalone document, while a Notion database is a structured collection of related pages with customizable properties and flexible visualization options.
